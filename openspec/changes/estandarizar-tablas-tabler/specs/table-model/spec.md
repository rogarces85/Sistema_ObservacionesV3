## ADDED Requirements

### Requirement: All tables use table-vcenter card-table table-hover
Every HTML `<table>` in the system SHALL have class `table table-vcenter card-table table-hover`.

#### Scenario: observaciones.php main table
- **WHEN** the observations table renders
- **THEN** it SHALL have `class="table table-vcenter card-table table-hover"` instead of only `id="observationsTable"` with no class

#### Scenario: usuarios.php table has hover
- **WHEN** the users table renders
- **THEN** it SHALL include `table-hover` in its class list

#### Scenario: establecimientos.php table has hover
- **WHEN** the establishments table renders
- **THEN** it SHALL include `table-hover` in its class list

#### Scenario: report tables have hover
- **WHEN** any of the 5 report tables render
- **THEN** they SHALL include `table-hover` in their class list

#### Scenario: supervision table uses table classes
- **WHEN** the supervision table renders
- **THEN** it SHALL have `class="table table-vcenter card-table table-hover"` and be wrapped in `div.table-responsive`

#### Scenario: eliminadas table uses table classes
- **WHEN** the deleted observations table renders
- **THEN** it SHALL have `class="table table-vcenter card-table table-hover"` and be wrapped in `div.table-responsive`

#### Scenario: JS-generated tables in asignaciones.php
- **WHEN** referentes table and reasignaciones table are generated by JavaScript
- **THEN** they SHALL use `class="table table-vcenter card-table table-hover"` in their HTML template strings

### Requirement: All tables wrapped in table-responsive div
Every `<table>` element SHALL be wrapped in a `<div class="table-responsive">` for horizontal scroll on mobile.

#### Scenario: observaciones.php has responsive wrapper
- **WHEN** the observations table renders
- **THEN** it SHALL be wrapped in `<div class="table-responsive">` instead of the current `<div class="overflow-x-auto">`

### Requirement: text-end replaces text-right
All table header cells with right alignment SHALL use `class="text-end"` instead of `class="text-right"`.

#### Scenario: usuarios acciones header
- **WHEN** the "Acciones" header renders
- **THEN** it SHALL use `class="text-end"` instead of `class="text-right"`

### Requirement: No Tailwind-like classes in table cells
Table cells SHALL NOT use `text-slate-*`, `font-bold`, `font-semibold`, `text-xs`, or `flex items-center gap-3` classes. They SHALL use Bootstrap/Tabler equivalents.

#### Scenario: observaciones table cells use Tabler classes
- **WHEN** observation rows render
- **THEN** cells SHALL use `fw-semibold`, `text-secondary`, `text-muted`, `small` instead of `text-slate-*`, `font-bold`, `text-xs`

### Requirement: Empty state uses consistent styling
Tables with no data SHALL display an empty state row with `text-center text-secondary py-4`.

#### Scenario: empty observations table
- **WHEN** there are no observations
- **THEN** the empty row SHALL use `class="text-center text-secondary py-4"`
