# Release 2.1.0 - Notas para usuarios

## Que cambia para mi?

Esta version refuerza la seguridad y consistencia del sistema,
sin cambios visibles en las pantallas del dia a dia. Aun asi,
hay dos diferencias que notaras:

### 1. Ventanas de confirmacion con el mismo estilo del sistema

Antes: aparecian ventanas pequenas del navegador ("Aceptar" /
"CANCELAR") para confirmar acciones delicadas.

Ahora: las confirmaciones usan el mismo diseno que el resto del
sistema (botones, colores, cabecera), con texto mas claro y, en
algunos casos, un campo para escribir una palabra de confirmacion.

Acciones que ahora piden confirmacion visual:
- Aprobar, cancelar o eliminar observaciones.
- Mover una observacion a la papelera.
- Restaurar o eliminar definitivamente desde la papelera.
- Restablecer la contrasena de un usuario.
- Eliminar un usuario.
- Remover una asignacion de un registrador.
- Copiar asignaciones del ano anterior.
- Crear o revertir un snapshot del sistema.

### 2. Restablecer contrasena

Si un supervisor te restablece la contrasena, ahora:

- Si el sistema tiene email configurado, recibes un correo con
  tu contrasena temporal.
- Si no, el supervisor la vera en pantalla y te la comunicara por
  el canal que usen. Pidele cambio inmediato desde tu perfil.

La contrasena temporal cumple la politica actual (12 caracteres,
mayuscula, numero y simbolo).

## Que quedo igual?

- Tu usuario, contrasena actual y datos de asignaciones.
- El flujo de creacion, edicion y eliminacion de observaciones.
- Los reportes, el dashboard y la papelera.
- Los filtros y busquedas.

## Que se corrigio en esta version?

Esta version no agrega funciones nuevas, sino que endurece las
existentes:

- Las acciones criticas ya no se pueden ejecutar por error
  haciendo doble clic rapido: cada accion se desactiva mientras
  esta en curso.
- Si una accion falla (por ejemplo, no hay conexion o no tienes
  permisos), veras un mensaje claro en vez de una pantalla en
  blanco.
- Los botones que confirman acciones destructivas son rojos y
  muestran el texto explicito ("Eliminar permanentemente",
  "Remover reasignacion", etc.).
- Los enlaces de descarga y aprobacion usan HTTPS obligatorio en
  produccion.
- Si iniciaste sesion, ya no podras cambiarte tu propio rol ni
  desactivarte a ti mismo desde la pantalla de Usuarios.

## Acciones que requieren atencion especial

- **Reset de contrasena**: si eres supervisor y restableces la
  contrasena de alguien, asegurate de que el usuario la cambie en
  su primer ingreso.
- **Eliminacion permanente en papelera**: solo se ejecuta si el
  supervisor tilda la casilla "Entiendo que esta accion no se
  puede deshacer". Esta proteccion es nueva y se aplica tanto en
  la pantalla como en la API.
- **Rollback de snapshots**: ademas de confirmar, ahora hay que
  escribir la palabra ACEPTAR para que el sistema continue.

## Preguntas frecuentes

**¿Mis contrasenas o datos cambian?**
No. Los usuarios, contrasenas y datos existentes se mantienen
exactamente como estaban.

**¿Debo hacer algo?**
No para el uso normal. Si te restablecen la contrasena, cambiala
de inmediato desde tu perfil (esquina superior derecha > Mi Perfil).

**Que pasa si mi navegador bloquea las nuevas ventanas?**
Si las modales de confirmacion no aparecen, prueba con Ctrl+F5
para recargar sin cache. Si el problema continua, contacta a
soporte.

**Como se si un supervisor elimino o restauro una observacion?**
El detalle de la observacion (boton "Ver detalle" en cada fila)
muestra el historial completo de cambios, con usuario, fecha y
comentario. La papelera tambien registra el motivo del cambio.

**Puedo seguir usando mis enlaces guardados?**
Los enlaces a pantallas (por ejemplo, ?page=observaciones) siguen
funcionando igual. Los enlaces a acciones especificas (como aprobar
o eliminar) ahora requieren la nueva confirmacion visual.
